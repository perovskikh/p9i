- Навигационная таблица README актуальна: все промпты зарегистрированы

## Когда использовать

- После создания нового промпта (проверить, что строка добавлена в README)
- После удаления промпта (проверить, что строка убрана)
- После обновления версии промпта (синхронизировать версию в таблице)
- Периодически (раз в неделю) для проверки актуальности реестра
- После массовых операций rename/delete в директории `docs/ai-agent-prompts/`

> **Важно:** Этот промпт обслуживает **только навигационную часть** README —
> таблицу в разделе «Шаблоны запросов» и записи в «Реестре версий».
> Он НЕ синхронизирует 6 разных секций с подробными описаниями (устаревший подход).
> Полная документация каждого промпта живёт ВНУТРИ файла промпта
> (секции `## Быстрый старт`, `## Когда использовать`, `## Журнал изменений`).

## Mission Statement

Ты — AI-агент, отвечающий за актуальность навигации в `docs/ai-agent-prompts/README.md`.
Твоя задача — сверить фактический список `.md` файлов в директории с записями в README,
синхронизировать версии и добавить/убрать строки при расхождении.
Главный принцип: README — это индекс-навигатор, не документация. Вся документация промпта
хранится в самом файле промпта.

## Шаг 0: Discovery

1. Получить список файлов в основной директории:
   ```bash
   ls docs/ai-agent-prompts/*.md | grep -v README
   ```
2. Получить список файлов в `meta-promptness/`:
   ```bash
   ls docs/ai-agent-prompts/meta-promptness/*.md
   ```
3. Прочитать текущее состояние навигационной таблицы в README.md
   (раздел `### 2) Операционные промпты` и `### 1) Meta-промпты`)
4. Построить diff:
   - Файлы есть в папке, но нет в таблице → нужно добавить
   - Строки в таблице, но нет файла → нужно убрать
   - Версия в таблице ≠ версия в файле → нужно обновить

## Шаг 2: Синхронизация навигационной таблицы

**Для каждого .md файла без строки в таблице:**
1. Прочитать заголовок файла (`head -5`) для определения версии и назначения
2. Добавить строку в формате:
   ```
   | Назначение | `promt-X.md` | «Используя `promt-X.md`, <краткий запрос>.» |
   ```
3. Разместить в правильную таблицу (Meta / Операционные)

**Для каждой строки без файла:**
1. Убрать строку из таблицы
2. Добавить запись в `## Журнал изменений README` с описанием удаления

**Для каждого файла с несовпадающей версией:**
1. Обновить версию в таблице по значению из файла (`grep "Версия:" file.md | head -1`)

**Порядок строк в таблицах:**
- Meta-промпты: `meta-promt-adr-system-generator` → `meta-promt-prompt-generation` → остальные
- Операционные: ADR-промпты → Code-промпты → Infra-промпты → Meta-промпты

## Шаг 3: Верификация

```bash
# Проверить количество .md файлов
echo "Файлов в директории:"
ls docs/ai-agent-prompts/*.md | grep -v README | wc -l
ls docs/ai-agent-prompts/meta-promptness/*.md | wc -l

# Проверить отсутствие ADR-XXX в README
echo "ADR-XXX нарушения (должно быть 0):"
grep -n "ADR-[0-9]\{3\}" docs/ai-agent-prompts/README.md | wc -l

# Проверить guard
make docs-prompt-guard
make docs-check-mkdocs-nav
```

Ожидаемый результат:
- Количество строк в таблице = количество .md файлов (без README и versioning-policy)
- `grep "ADR-[0-9]\{3\}"` возвращает 0 строк
- Guards проходят без ошибок

## Anti-patterns

1. **Добавление подробных описаний в README** вместо в файл промпта
2. **Создание 6+ секций** для каждого промпта в README (устаревший подход)
3. **Изменение версии в таблице** без проверки актуальной версии в файле
4. **Добавление ADR-XXX номеров** в строки таблицы
5. **Редактирование файлов промптов** в рамках этого промпта (только README)
