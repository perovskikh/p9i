# promt-prompt-creator

## Назначение
Создание нового промта для ai-prompt-system по описанию на естественном языке.
Это мета-промт — промт для создания промтов (Prompt Engineering).

## Когда использовать
- "создай промт для ..."
- "добавь в систему промт который ..."
- "нужен шаблон для ..."
- "напиши prompt для ..."

## Входные данные
- `idea` — описание что должен делать новый промт (обязательно)
- `name` — желаемое имя файла, без префикса (опционально)
- `category` — тип: feature / fix / docs / test / infra (опционально)

## Алгоритм выполнения

### Шаг 1 — Анализ намерения
Определи:
- Что промт должен ДЕЛАТЬ (глагол действия)
- Что получает НА ВХОДЕ (контекст, файлы, параметры)
- Что отдаёт НА ВЫХОДЕ (код, текст, план, анализ)
- Какие ОГРАНИЧЕНИЯ и ПРАВИЛА важны

### Шаг 2 — Выбор имени файла
Формат: `promt-{глагол}-{объект}.md`
Примеры:
- promt-security-audit.md
- promt-readme-sync.md
- promt-db-migration.md

### Шаг 3 — Генерация промта
Создай файл со структурой:

```markdown
# promt-{name}

## Назначение
{Одно предложение — что делает промт}

## Когда использовать
- {триггерные фразы для ai_prompts маршрутизатора}

## Входные данные
- `param1` — описание (обязательно/опционально)

## Алгоритм выполнения
### Шаг 1 — {название}
{инструкции}

### Шаг 2 — {название}
{инструкции}

## Выходной формат
{описание ожидаемого результата}

## Примеры
### Пример 1
Вход: ...
Выход: ...

## Ограничения
- {что промт НЕ делает}
```

### Шаг 4 — Верификация (Self-check)
Проверь созданный промт:
- [ ] Намерение понятно за 5 секунд?
- [ ] Входные параметры задокументированы?
- [ ] Есть пример использования?
- [ ] Нет конфликта с существующими промтами?
- [ ] Триггерные слова добавлены в секцию "Когда использовать"?

### Шаг 5 — Добавление маршрутизации
Выведи строку для добавления в `ai_prompts` маршрутизатор:

```python
# В src/api/server.py:
"{keyword1}, {keyword2}": "promt-{name}"
```

### Шаг 6 — Сохранение
Создай файл: `prompts/promt-{name}.md`
Выведи подтверждение с путём и кратким описанием.

## Выходной формат
1. Готовый `.md` файл промта
2. Строка маршрутизации для `ai_prompts`
3. Команда для теста: `use ai-prompts` + пример запроса

## Ограничения
- Не создаёт дублирующие промты (проверяй список существующих)
- Имя файла всегда с префиксом `promt-`
- Промт должен быть самодостаточным — без зависимостей от других промтов
